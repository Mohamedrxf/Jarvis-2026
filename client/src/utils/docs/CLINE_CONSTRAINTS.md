# Cline Development Rules

1. Reuse existing architecture.
2. Search existing code before creating new code.
3. Never rewrite working modules.
4. Modify the minimum number of files.
5. Never create duplicate logic.
6. Never create duplicate services.
7. Never generate documentation unless requested.
8. Never generate long explanations.
9. Return concise output only.
10. Do not generate large reports.
11. Do not create diagrams.
12. Do not create markdown summaries.
13. Do not explain code unless requested.
14. Implement only the requested feature.
15. Verify only the requested feature.
16. Fix only runtime bugs.
17. Preserve backward compatibility.
18. Use existing coding style.
19. Stop immediately after the requested task completes.
20. Never continue to the next phase automatically.
